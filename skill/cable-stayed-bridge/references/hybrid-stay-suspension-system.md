# 扩展模块:斜拉-悬吊协作体系(斜拉+悬索混合体系)

本模块基于真实工程蓝本(双主缆-斜拉协作体系,边跨同步施工、先边跨合龙后中跨合龙)提炼。这类体系在中跨/边跨同时存在主缆+吊索(悬索桥特征)和斜拉索(斜拉桥特征),施工上通常表现为"顶推架设主缆和索鞍 + 边跨/中跨吊索与斜拉索交替张拉 + 多次顶推调整索鞍位置 + 先边跨合龙、后中跨合龙"。这是比标准斜拉桥或标准悬索桥都复杂的特种体系,遇到时按下面的差异点核对,不要照搬标准斜拉桥流程。

## 判断是否需要用这个模块的快速标志

用户描述里如果出现以下任意关键词组合,大概率是协作体系,请调用本模块:

- "主缆"+"斜拉索"同时出现
- "索鞍顶推"、"顶推力"多次编号(1~9甚至更多)
- "先边跨合龙,后中跨合龙"或类似表述
- "悬吊区"+"斜拉区"分开命名的边界组
- "交叉索"这个构件名

## 与标准斜拉桥的关键差异

### 1. 多了"主缆架设"和"顶推"两类专属工序

阶段骨架(节选,已按逻辑分组,实际每一步都是独立施工阶段):

```
塔墩施工              → 建塔、墩;塔墩底边界;自重 + 塔顶补偿荷载
主缆架设              → 激活主缆单元;锚点、塔-索鞍边界(DEFORMED)
顶推装置              → 激活"刚臂""顶推杆"施工临时构件;加索鞍顶推荷载
边梁塔梁架设           → 激活塔区/边墩区梁段;各类临时支撑(DEFORMED)
索夹安装              → 加索夹专项荷载(模拟索夹自重/预紧对主缆的作用)
[循环 N 次] 斜拉-悬吊同时施工i → 每次同步激活: 拉索对i + 对应钢梁对i + 若干边跨/跨中吊梁、吊索
                                  同时把已合龙区域的临时铰接/受压约束转DEFORMED,未合龙的留ORIGINAL
第i次顶推             → 加顶推力i荷载(逐级顶推索鞍/主缆张力调整,i通常到9~15次不等)
斜拉桥合拢            → 最后一对拉索+梁段合龙
临时索                → 激活边跨临时索(合龙前的过渡受力体系);同时钝化被替代的边跨吊索
边跨合拢              → 激活边跨最后几个吊梁/吊索;撤除临时索
二次调索              → 激活"二次调索"专项工况;所有拉索对整体钝化重建(内力/应力状态重分布),用于消除合龙误差、二次精调索力
跨中悬吊区施工         → 补建剩余跨中吊梁/吊索
重叠区临时索           → 中跨合龙前的过渡索
跨中合拢              → 激活中跨合龙索、合龙梁段
交叉索施工             → 激活主缆与斜拉索交叉区的"交叉索"单元(协作体系特有的传力构造);撤除重叠区临时索
铰接转固结             → 统一把全桥悬吊区、边跨悬吊区、斜拉区的临时铰接/受压约束撤除,转固结;同时撤除各类临时支撑
二恒                  → 撤除刚臂/顶推杆等施工设备;加二期恒载
```

**要点**:这个体系是"先边跨合龙,后中跨合龙"——如果用户的协作体系是反过来的(先中跨合龙),阶段顺序要镜像调整,不能照搬。

### 2. 材料要多建两类

除标准斜拉桥的塔/墩/主梁/斜拉索材料外,协作体系还需要:

- **主缆专用钢材**(高强钢绞线束,弹性模量取有效模量)
- **吊索钢材**(和斜拉索、主缆分开建,阻尼比/弹模都可能不同)
- **顶推装置**:自定义材料,给一个很大的弹性模量模拟施工期刚性顶推构件,这个材料只在施工阶段用,合龙后要整体删除对应单元组

### 3. 分组要按"张拉批次+空间位置"双重维度命名

- `拉索对NN`:第 NN 批张拉的斜拉索(塔两侧对称各一根,共2根一组);注意有的工程是从跨中往塔方向倒序编号,和常规从塔往跨中顺序编号相反,建模时要跟用户确认实际编号习惯
- `钢梁对NN`:第 NN 批架设的主梁节段(和拉索对同步)
- `跨中吊梁NN` / `跨中吊索对NN`:中跨区吊索/吊梁,独立于斜拉索批次编号
- `边跨吊梁NN` / `边跨吊索NN`:边跨区吊索/吊梁,单独编号
- `交叉索N`:主缆与斜拉索交叉区传力索,合龙后期才激活
- 每一类边界(铰接、受压、固结)都按区域分开:`悬吊区铰接N`/`悬吊区受压N`(中跨)、`边跨悬吊区铰接N`/`边跨悬吊区受压N`(边跨)、`斜拉区铰接N`/`斜拉区受压N`(斜拉区)——**三个区域的临时约束不能混用同一组**,因为它们的撤除时机不同

### 4. "二次调索"是这类体系的标配步骤,不要漏掉

悬索桥+斜拉桥协作受力,合龙后索力会因为边跨/中跨顺序合龙产生偏差,标准做法是单独设一个施工阶段:先把所有拉索对整体钝化(按内力重分布方式,不是真的删除单元而是重建),再重新激活一次做二次调索,用来修正成桥索力。这是唯一一个"先卸后重建同一批单元"的特殊写法,和其余"一次性激活、合龙后再撤除"的逻辑不同,容易漏掉。

### 5. 顶推荷载和顶推杆/刚臂是纯施工期虚拟构件

`刚臂`、`顶推杆`两个单元组,以及每次的顶推力荷载组,只在施工阶段起作用(模拟索鞍顶推工装),在最后"二恒"阶段要整体撤除,不能带入运营阶段模型,否则会残留虚假刚度。
