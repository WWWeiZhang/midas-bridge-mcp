# 施工阶段(阶段二)工作流:命名与分组模板

**前提**:阶段一(成桥分析)已完成,手上已经有成桥索力表和成桥内力/线形结果。本文档只覆盖标准悬臂拼装斜拉桥;如果是主缆+斜拉索+吊索的协作体系,或涉及顶推、边跨同步施工,先看 `hybrid-stay-suspension-system.md`。

对应 MCP 工具模块:

- `group_tools`:建结构组/边界组/荷载组(施工阶段激活/钝化的最小单元)
- `boundary_tools`:临时支撑、临时铰接约束
- `construction_stage_tools`:定义每个施工阶段(激活/钝化单元、边界、荷载)
- `tendon_tools`:每个阶段的张拉控制值(从阶段一的成桥索力反推无应力索长/张拉力)
- `analysis_control_tools` + `analysis_tools`:施工阶段非线性分析设置与运行
- `result_tools`:每阶段索力/内力/线形核查,以及最终成桥态是否收敛回阶段一的目标值

这是从蓝本工程抽象出的通用命名模板。给一座新的斜拉桥建模时,把下表里的 `N`/`NN` 替换成实际的悬臂节段/张拉批次数量,直接套用即可,不用每次重新设计命名规则。

## 单元组(GROUP)命名模板

| 组名模式 | 含义 | 建议编号方向 |
|---|---|---|
| `塔` | 全部塔柱单元 | 一次性建组,不分批 |
| `墩` | 全部墩身/承台单元 | 一次性建组 |
| `拉索对NN` | 第 NN 批张拉的一对(或一组)斜拉索 | 从塔根部(NN小)到跨中(NN大),或反过来,与用户确认后固定一种,全文档统一 |
| `钢梁对NN` / `梁段NN` | 第 NN 批架设的主梁节段,与`拉索对NN`同步编号 | 同上 |
| `边墩梁-左/右` | 边跨端部与边墩相连的梁段 | 单独命名,不参与主编号 |
| `临时支撑NN` | 悬臂施工期承托未合龙梁段的临时支墩/支架 | 按位置命名(左边跨/右边跨/塔区) |
| `合拢段` / `跨中合拢梁` | 最终合龙节段 | 单独命名 |
| `二次调索` | 合龙后整体重新张拉索力的专项工况单元组 | 仅协作体系/大跨度斜拉桥需要 |

## 边界组(BNDR-GROUP)命名模板

| 组名模式 | 含义 | 何时激活 | 何时撤除 |
|---|---|---|---|
| `塔墩底` | 塔/墩与地基的固结 | 塔墩施工阶段,一次性,不撤除 | — |
| `拉索-梁对NN` | 第 NN 批拉索与梁的锚固连接 | 对应梁段+索被激活的同一阶段 | 通常不撤除(永久连接) |
| `塔-梁` | 塔梁固结或支座连接 | 视体系(固结体系 vs 漂浮体系)在梁段到达塔位时激活 | 半漂浮体系合龙后可能需要调整支座类型,视具体设计 |
| `XX区临时支撑` | 悬臂期间的临时支墩 | 相邻梁段架设时激活 | 该区域合龙、体系自持后撤除 |
| `XX区铰接NN` / `XX区受压NN` | 节段接头处的临时铰接+仅受压约束(悬臂拼装常见做法) | 该节段架设时激活 | 全桥合龙、体系转换阶段统一撤除,转固结 |

## 荷载组(LOAD-GROUP)命名模板

| 组名模式 | 含义 | 施加时机 |
|---|---|---|
| `G` | 结构自重 | 每个构件在被激活的同一阶段随激活单元一起触发,或单独一次性施加 |
| `二期` | 桥面铺装、护栏等二期恒载 | 全桥合龙+体系转换完成之后,最后一个阶段 |
| `压重NN` | 悬臂施工配重(平衡弯矩) | 对应悬臂节段架设阶段 |
| `顶推力NN` | 顶推施工力(如有顶推工艺) | 每次顶推对应一个独立阶段 |
| `二次调索` | 合龙后的索力精调 | 合龙、二次调索阶段 |

## 通用施工阶段骨架(标准悬臂拼装斜拉桥,无顶推)

```
1. 塔墩施工
2. 塔身各节段施工(可合并为一个阶段,或按提升段拆分)
3. 起始梁段/合龙前梁段架设(通常在塔根部,支架现浇或临时支撑)
4. 循环 N 次:
   第i节段架设 + 第i对索张拉
   → 激活: 钢梁对i, 拉索对i
   → 激活边界: 拉索-梁对i(DEFORMED), 悬臂区铰接i(DEFORMED)
5. 合龙段架设(边跨合龙 / 中跨合龙,视方法而定顺序)
6. 二次调索(可选,大跨度/协作体系建议做)
7. 铰接转固结(把所有临时铰接约束批量撤除,转永久固结/支座)
8. 撤除全部施工临时构件(支架、临时索、顶推装置等)
9. 二期恒载
```

如果施工方法是"边跨同步施工、先边后中合拢"或涉及主缆/吊索的协作体系,改用 `hybrid-stay-suspension-system.md` 里的骨架。

## 施工阶段激活/钝化的语义(ORIGINAL vs DEFORMED)

激活边界时要指定基准状态:

- **DEFORMED**:在当前已变形的几何位置上建立边界,不产生附加内力(常规做法,绝大多数施工阶段激活都用这个)
- **ORIGINAL**:按初始几何位置建立边界,会引入几何修正/附加变形——只在有特殊设计意图(预抬高、预偏位)时才用

钝化单元时,通常代表"按 100% 内力重分布方式移除"(不是物理删除,是把该单元当前承担的内力释放并重新分配给剩余结构)。
