# Midas Civil MCP —— 钢结构建模

介绍如何利用 **Midas Civil MCP** 工具建立并分析钢结构模型。

默认假设 **MCP Server 已成功连接**。

> **重要提示**：本文档中给出的工具名和参数结构是常见用法示例，帮助你快速理解整体流程。
> 实际调用前，请始终以当前 MCP 工具列表（tool schema）中的定义为准 —— 工具可能已迭代，
> 字段名或结构如与本文档有出入，以 schema 为准。

---

# 开始之前

首先始终调用：

```text
get_all_nodes
```

检查当前模型是否为空。

如果发现模型中已经存在节点（说明是之前建立过的模型），应提醒用户：

> 当前模型检测到已有数据（共 N 个节点）。
> 请说明是希望：
> 1) 保留现有模型并在其基础上继续建模；
> 2) 覆盖 / 清空现有数据后重新开始（可在 Midas Civil NX 中按 **Ctrl+N** 新建工程）；
> 3) 还是这本身就是你之前正在编辑的模型，无需处理。
> 在收到用户明确回复前，不要执行任何删除或覆盖操作。

---

# 建模路径选择

在开始建几何之前，先判断这是哪一类结构，因为两类结构的建模逻辑差别很大：

**走"简单正交框架"路径（继续读本文档下面的内容）**，如果结构是：

- 平屋面 / 单坡屋面，仅由竖向柱和水平梁组成；
- 没有提到桁架、腹杆、弦杆、人字形屋顶、双坡屋面；
- 没有提到夹层楼面、纵墙支撑桁架、山墙端跨专门加撑。

**走"桁架屋面工业厂房"路径**，如果用户提到以下任意特征：

- 人字形 / 双坡 / 折线形屋面，屋脊比檐口高；
- 屋面或侧墙由"桁架"构成（弦杆 + 腹杆），而不是简单梁；
- 有夹层 / 中间楼层楼面；
- 纵向侧墙设置水平支撑桁架（而不是普通连系梁）；
- 只在两端开间设置十字交叉支撑（"端跨支撑"）。

满足桁架屋面特征时，**先读取并遵循** `references/truss-roof-building.md` 中的完整流程，
不要直接套用下面的简单框架步骤 —— 桁架结构需要的工具能力（桁架单元、空间框选、
分组查询）和建模顺序都不同。

# 单位说明

Midas Civil 默认单位为：

- 力：**tonf**
- 长度：**m**
- 位移：**m**

如果用户提供的荷载单位为 **kN**，需要提醒用户进行换算：

```text
1 tonf ≈ 9.81 kN
```

例如：

```text
3 kN ≈ 0.306 tonf
```

**材料密度字段的特殊单位**：材料定义中的 Weight Density 和 Mass Density 两个字段，
即使全局力/长度单位是 tonf/m，也固定按以下单位输入（这是软件本身的约定，不是笔误）：

| 字段 | 单位 | 常用钢材数值 |
|------|------|------|
| Weight Density（容重） | kN/m³ | 78.5 |
| Mass Density（质量密度） | ton/m³ | 7.85 |

**英制单位（kips / ft）**：如果用户描述的尺寸和荷载是英制（如 ft、kips），
**不要**默认套用 tonf/m 假设。需要先确认 MCP 是否提供设置单位体系的工具
（对应 Midas 里的 `Model.units(...)`），并在建几何**之前**完成单位设置 ——
单位体系需要在建模开始时一次性确定，中途切换容易导致已建部分的数值含义错乱。

# 建模总体流程（简单正交框架）

必须严格按照以下顺序：

1. 几何
2. 材料
3. 截面
4. 单元
5. 支座
6. 荷载
7. 分析

原因：

- 节点不存在，无法建立单元；
- 单元不存在，无法施加荷载；
- 荷载工况不存在，也无法加载。

其中第 2 步（材料）和第 3 步（截面）互不依赖，也不依赖节点，可以和节点创建
在同一轮回复中一起发起调用，无需等待前一个工具返回结果后再调用下一个。

---

# 1. 建立几何（先规划，不急于调用 MCP）

首先确定结构网格：

- **X** —— 建筑纵向
- **Y** —— 建筑横向
- **Z** —— 竖向（高度）

明确：

- X 方向共有多少轴线？
- Y 方向共有多少轴线？
- 每个方向的间距是多少？

建议采用规则编号，方便后续建立单元。

例如，一个：

- 两跨
- 每跨 4 m
- 建筑宽度 6 m
- 层高 3 m

的钢框架：

```text
Ground (Z=0)

1(0,0,0)   2(4,0,0)   3(8,0,0)

4(0,6,0)   5(4,6,0)   6(8,6,0)


Roof (Z=3)

7(0,0,3)   8(4,0,3)   9(8,0,3)

10(0,6,3) 11(4,6,3) 12(8,6,3)
```

始终保持这一节点空间布局，以便后续建立单元。

---

# 2. 批量创建节点、材料和截面

这三项可以并行完成（同一轮内一起调用）。

## 节点

尽量一次性调用 `create_nodes`，批量创建全部节点，而不要逐个创建。

参数格式示例：

```json
{
  "nodes": [
    {"id": 1, "x": 0, "y": 0, "z": 0},
    {"id": 2, "x": 4, "y": 0, "z": 0},
    {"id": 3, "x": 8, "y": 0, "z": 0}
  ]
}
```

**大模型提示**：当节点数量较大（约 4 万个以上）时，工具内部会自动分批发送请求，
这是正常行为，不代表调用出错。如果返回结果里出现部分批次失败，应先向用户说明
哪些节点 ID 未创建成功，再决定是否重试。

## 材料

优先调用 `create_steel_material`（数据库材料）。

参数格式示例：

```json
{
  "id": 1,
  "standard": "ASTM(S)",
  "name": "A36"
}
```

如果数据库检索失败，则立即改用 `create_user_steel_material`：

```json
{
  "id": 1,
  "name": "Custom-A36",
  "elastic_modulus": 200000000,
  "poisson_ratio": 0.30,
  "weight_density": 78.5,
  "mass_density": 7.85
}
```

参数推荐（不确定单位时参考"单位说明"章节）：

| 参数 | 数值 |
|------|------|
| E | 200000000 kPa |
| ν | 0.30 |
| Weight Density | 78.5 kN/m³ |
| Mass Density | 7.85 ton/m³ |

## 截面

调用 `create_db_section`，建议使用 AISC H 型钢。

参数格式示例：

```json
{
  "id": 1,
  "type": "H",
  "name": "W10X49"
}
```

默认可采用：

- 柱：**W10×49**
- 梁：**W16×67**

如果用户没有指定截面，应自行选择合理尺寸，并向用户说明所采用的截面。

---

# 3. 建立梁单元

先在本地整理完整单元列表，再一次性调用 `create_beam_elements`。

参数格式示例：

```json
{
  "elements": [
    {"id": 1, "node_i": 1, "node_j": 7, "material_id": 1, "section_id": 1},
    {"id": 2, "node_i": 2, "node_j": 8, "material_id": 1, "section_id": 1}
  ]
}
```

`node_i` / `node_j` 分别对应单元的起点和终点节点 ID，实际字段名请以工具 schema 为准。

建议按以下顺序编号：

1. 柱（竖向构件）
2. 横向主梁（连接两侧柱）
3. 纵向梁（沿建筑长度方向）
4. 支撑（最后建立）

对于规则钢框架：

- 柱：底节点 → 顶节点
- 横梁：连接同一 X 坐标下左右柱顶
- 纵梁：连接相邻柱顶

---

# 4. 屋面水平支撑

仅有正交梁柱的钢框架在水平方向并不稳定，必须设置屋面 X 型支撑。

推荐连接方式：

```text
node(i, 左侧)
      \
       \
        \
      node(i+1, 右侧)
```

形成 **X** 型布置，可有效防止屋盖发生侧向剪切变形（Racking）。

建议：

- 使用较小截面；
- 两端释放 **My**、**Mz**（见第 6 步）；

因为支撑应按桁架杆件工作，而不是刚接梁。

---

# 5. 支座

所有底层柱脚均采用固定支座。

调用 `create_supports`，参数格式示例：

```json
{
  "node_ids": [1, 2, 3, 4, 5, 6],
  "type": "fix"
}
```

一次性作用于所有底层节点，`node_ids` 传入对应的节点 ID 列表即可，不需要逐个节点调用。

---

# 6. 梁端释放

屋面斜撑应释放：

- My
- Mz

两端均释放。

调用 `create_beam_end_release`，参数格式示例：

```json
{
  "releases": [
    {"element_id": 21, "i_end": ["My", "Mz"], "j_end": ["My", "Mz"]}
  ]
}
```

具体字段名（如自由度是用字符串数组还是布尔组合）请以工具 schema 为准；
如实测格式不同，按实际返回的报错信息调整后重试。

柱和主梁通常保持刚接，不进行释放；除非用户明确要求采用铰接连接。

---

# 7. 建立荷载工况并施加荷载

必须先创建荷载工况，再施加荷载。

典型工业厂房可采用：

| 工况 | 类型 | 内容 |
|------|------|------|
| DL-SelfWeight | D | 自重，Z = -1 |
| DL-Floor | D | 屋面梁均布荷载，GZ，约 0.3 tonf/m（≈3 kN/m） |
| WL-WindX | W | 顶部柱节点 X 向风荷载 |
| WL-WindY | W | 侧框架梁柱 GY 向均布风荷载 |

调用荷载工具时需要指定该荷载所属的工况名称（如 `"case": "DL-Floor"`），
不同工况之间互相独立，后续提取结果时也需要按工况分别获取（见第 8 步）。

说明：荷载 ID 覆盖问题已修复，现在可安全地多次调用均布荷载接口，系统会
自动递增内部荷载编号。

---

# 8. 运行分析并输出结果

调用：

```text
run_static_analysis
```

随后针对**每一个荷载工况**分别调用：

```text
get_nodal_displacements  (参数中指定 case，如 "DL-SelfWeight")
get_reactions            (参数中指定 case，如 "WL-WindX")
```

不要只取默认工况的结果 —— 本技能第 7 步定义了多个工况（自重、活载、两个方向风载），
需要逐一提取，否则会遗漏关键结果（例如风荷载下的最大侧移）。

汇报时重点说明：

- 每个工况下的最大位移及其位置；
- 总支座反力是否与该工况总外荷载平衡；
- 是否存在分析警告及其影响。

---

# 常见问题

## 某个荷载工况结果全为零

说明荷载在会话过程中丢失。

解决方法：重新施加荷载，并重新运行分析。

---

## "几何形状不正确"

通常表示存在零长度单元。

检查：两端节点是否坐标完全相同。

---

## RX 奇异性（Singularity）警告

若模型采用仅受轴力的支撑杆件（桁架/斜撑），出现 RX 奇异性通常属于正常现象。

一般不会影响：

- 竖向位移；
- 轴力计算。

可忽略该警告。桁架屋面结构里这类警告会更常见，详见
`references/truss-roof-building.md` 中的说明。

---

## 材料不存在

数据库材料查找失败时，应立即改用 `create_user_steel_material` 建立自定义钢材。

---

## 模型中已有旧数据

通过 `get_all_nodes` 若发现存在意外节点，应提醒用户当前模型非空，
并按"开始之前"章节的流程，等待用户明确回复（保留 / 覆盖 / 确认是同一模型）
后再继续操作，不要自行删除或覆盖已有数据。
