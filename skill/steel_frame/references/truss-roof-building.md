# 桁架屋面工业厂房建模

适用于：长向多榀重复框架的预制钢结构厂房，特征包括双坡/人字形屋面桁架、
纵墙水平支撑桁架、山墙端跨十字撑、多层夹层楼面。

这类结构和"简单正交框架"的本质区别是：屋面和侧墙的很多构件是**按桁架方式
受力**（弦杆承受弯矩/轴力、腹杆只受轴力且两端铰接），而不是刚接的梁柱体系。
建模顺序、需要的工具能力都不一样，不要直接套用简单框架的步骤。

---

## 建模前必须确认：MCP 工具是否支持以下能力

在开始之前，先确认当前 MCP 工具列表里是否有对应能力。如果缺失，需要先和
用户/工具开发者沟通补齐，而不是勉强用现有工具拼凑（容易建错几何或漏掉铰接）。

| 需要的能力 | 用途 | 缺失时的影响 |
|---|---|---|
| 创建**桁架单元**（只受轴力，两端自动释放弯矩） | 屋面桁架腹杆、纵墙支撑桁架、端跨十字撑 | 若只能用梁单元硬释放弯矩代替，容易漏做或做错释放方向 |
| **空间范围框选**节点/单元（按坐标 Box 选取） | 选出所有底层节点做支座、选出某面墙的单元加荷载 | 只能靠人工列节点 ID，规模大时极易出错或遗漏 |
| **按分组标记 + 查询**单元/节点（打标签后可整组取出） | 先建出一排构件，之后按组取节点列表做二次连接（如腹杆连弦杆） | 无法做批量的"等分点两两连接"这类操作 |
| **起点+方向+长度+等分数**的单元批量生成 | 一次性生成一排等分节点和梁（檩条、桁架弦杆常见需求） | 只能逐个算坐标建节点，工作量成倍增加 |

如果这些能力目前 MCP 里没有，应如实告知用户"当前工具集不支持桁架结构建模所需的
某些操作，建议先扩展 MCP 再继续"，而不是尝试用简单框架的工具勉强拼出桁架效果。

## 参数规划

建这类结构前，先和简单框架一样明确网格参数，但这里的参数更多、含义也不同：

| 参数 | 含义 |
|---|---|
| 开间数（榀数 - 1） | 沿建筑长度方向重复框架的跨数 |
| 开间长度 | 相邻两榀框架之间的纵向距离 |
| 檐口高度 | 柱顶到屋面桁架下弦的高度 |
| 建筑跨度（横向总宽） | 两侧外柱之间的总宽度，若有纵向中线立柱，指其到两侧的距离之和 |
| 屋脊起坡高度 | 屋脊比檐口高出的高度（决定双坡屋面的坡度） |
| 桁架腹杆等分数 | 屋面桁架和纵墙支撑桁架上，弦杆之间要分成几段来布置腹杆 |
| 桁架下弦离檐口的距离 | 有些设计里下弦和上弦之间留一段净空（对应檐口梁往上先接一段再到桁架） |
| 夹层层数 / 分层数 | 檐口以下要设置几层夹层楼面 |

先向用户确认这些参数（如果用户只给了大致描述，按常见工业厂房比例推算并说明假设），
再进入下面的建模步骤。

## 建模步骤

### 1. 主体柱列与檐口梁

每榀框架两侧（以及纵向中线，如果有）先建竖向柱，从地面到檐口高度。
如果桁架下弦和檐口之间有一段净空，柱子可以分两段建：地面到"檐口-净空高度"、
再到檐口高度，方便后续分别指定不同的释放条件。

### 2. 屋面双坡桁架（先建单榀，再复制到其他榀）

一榀桁架通常由这几部分组成：

- **上弦**：从檐口沿屋面坡度方向，向内延伸到屋脊下方的中线位置，形成两段
  （对应两个坡面）。上弦按刚接梁建立，用等分数确定节点间距，方便后续连接腹杆。
- **脊柱**（如有）：屋脊处一根竖杆，连接两侧上弦的交点和下弦，代表屋脊的
  起坡高度。
- **下弦**：沿檐口高度水平直线连接两侧，同样按等分数布置节点。
- **腹杆**：按上弦和下弦对应的等分点，两两连接形成三角形，腹杆用**桁架单元**
  （两端铰接、只受轴力），不要用刚接梁代替。

建好一榀的桁架几何后，其余榀按开间长度整体平移即可，不需要重新推导坐标。

### 3. 山墙端跨十字撑

只在**第一跨和最后一跨**（即两端山墙所在的开间）的山墙面和屋面设置交叉支撑，
用来提供整体纵向稳定性，中间各跨不需要重复设置。支撑同样用桁架单元。

### 4. 纵墙水平支撑桁架

沿建筑长度方向，在檐口高度设上下两道弦杆，中间用斜腹杆连接，形成侧墙的
**水平桁架**（不是简单连系梁），用来抵抗纵向荷载（如纵向风荷载、地震作用）。
弦杆可以是梁单元，腹杆用桁架单元。

### 5. 夹层楼面

在檐口以下，按需要的层数把高度等分，每层设置外侧和中线（如有）两道次梁，
再横向连接形成楼面梁格。次梁通常用刚接梁单元。

### 6. 支座

底层柱脚全部固定。用空间框选工具一次性选出所有 Z = 0（地面标高）的节点，
避免手动罗列。

## 荷载

- **自重**：调用自重相关工具自动施加，不需要手动计算。
- **山墙面风荷载**：只对山墙范围内的节点施加**节点集中力**（沿 Y 方向，即
  垂直于山墙面的方向），不要按梁均布荷载处理 —— 山墙风压通常按节点力简化。
- **侧墙面风荷载**：对侧墙范围内的梁单元施加**沿局部坐标方向的均布荷载**
  （沿局部 X 方向，通常记作 GX 或类似的局部坐标标识），需要先用框选工具选出
  侧墙范围内的梁单元 ID，再统一施加。

## 分组与节点顺序的重要提醒

如果使用"先建一整排构件、打分组标签，之后按组取节点列表、按等分点顺序两两连接"
的方式来生成腹杆或次梁，**必须先验证分组查询返回的节点顺序是否和几何创建顺序
一致**，否则很容易把两个本该相邻的等分点连成交叉或跳跃的错误杆件。

建议做法：建完一小段后，先用 `get_all_nodes` 核对几个关键节点的坐标，确认
连接逻辑符合预期（比如腹杆确实连的是相邻的上下弦节点），再批量建立剩余部分。

## 常见问题（桁架结构特有）

### RX 奇异性警告非常多

桁架体系里大量构件只受轴力、两端铰接，分析时出现较多 RX（绕杆轴扭转）方向的
奇异性警告是正常现象，不影响竖向位移和轴力结果，可以忽略。如果同时伴随位移
异常大或分析直接失败，才需要进一步排查（通常是某处几何没连上，导致局部机构）。

### 空间框选返回空列表

检查框选范围的两个角点坐标是否真的把目标节点/单元包含在内（注意是否为
闭区间、两个角点谁大谁小是否重要，取决于工具实现），可以先用较大的范围
框选做验证，缩小范围再精确框选。

### 单位不是默认的 tonf/m

如果用户的尺寸和荷载是英制（kips、ft）等非默认单位，务必在**建几何之前**
先设置好单位体系，不要中途切换，否则已建部分的数值含义会和新建部分不一致。
详见主 SKILL.md 中"单位说明"章节。
