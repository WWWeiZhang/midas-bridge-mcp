# 精细板/壳箱梁模型——带实际开孔几何形状

当模型需要表示箱形截面腹板中圆形（或其他形状）开孔的**实际几何形状**时使用此工作流程——而非折减截面近似——通常在开孔周围带有加劲环，由板/壳单元构建，并通过刚性连接连接到梁单元支座。

这与主 SKILL.md 中的简化梁路径是根本不同的建模范式：几何形状由许多通过放样和拉伸生成的小板单元构建，而非少数几个梁段。

---

## 开始前确认 MCP 支持这些功能

此工作流程需要板/壳专用能力。检查工具列表中是否包含：

| 能力 | 用途 |
|---|---|
| 创建**板单元**（不仅仅是梁） | 所有壳几何形状：孔周围环、腹板、翼缘、加劲肋 |
| 在两个节点组之间**放样**（生成连接两条曲线的板环/板带） | 构建内孔边界与外加劲边界之间的环形板 |
| 沿某个方向**拉伸**一条节点线并分段 | 从现有边生成板带——反复用于从开孔向外构建腹板、翼缘和加劲板 |
| **厚度**定义，与梁截面分开 | 板单元引用厚度 ID，而非截面 ID——不同的板（主板、薄加劲肋、腹板、翼缘）需要不同的厚度 ID |
| **刚性连接/主从**约束 | 将壳模型的横截面连接到单个节点，以便梁单元可以连接到它（见下面的"壳到梁过渡"） |
| **框选/线选**以通过几何条件检索节点或单元 ID | 选择要拉伸的边或面上的节点，或施加荷载/刚性连接，而无需手动列出 ID |

如果以上任何一项缺失，请直接告知用户，而不是尝试用梁单元近似实际开孔几何形状——那将违背选择此路径的初衷。

---

## 几何构建模式

构建带孔腹板的整体方法是：**围绕开孔构建一圈节点，然后反复向外拉伸/放样**以逐层生长板网格，直到完整的加劲开孔被建模完成。这比逐个放置每个节点高效得多。

### 1. 生成圆形节点环

使用基本三角函数计算围绕圆的节点坐标（围绕开孔以 `360° / n` 为角度步长），然后创建这些节点并标记组名（例如 `Circle_I` 用于内环）。这部分是纯粹的坐标计算——除了节点创建外不需要特殊工具。

### 2. 构建偏移外边界

创建第二圈节点，从圆向外偏移（遵循圆角矩形或开孔周围的类似边界形状），使用起点 + 终点 + 分段数的节点生成工具。将此组分开放置（例如 `Circle_O`）。

### 3. 在两个环之间放样以生成环形区域板

使用内外节点组之间的放样工具直接在开孔周围生成板单元环（开孔边缘与加劲边界之间的"垫圈"材料）。这是加强开孔的板厚度。

### 4. 反复向外拉伸以构建截面的其余部分

从外边界开始，沿相关方向（厚度方向、沿梁方向、横向）拉伸，以构建：

- 顶部和底部翼缘相邻板
- 侧面腹板
- 加劲板，通常作为**对称对**——从同一起始边在 +Y 和 -Y 方向各拉伸一次以镜像几何形状，而不是独立构建每侧

每次拉伸调用需要：源边（作为节点选择）、方向向量、分段数、环是否应闭合、以及生成的板应使用的厚度 ID。

### 5. 在远端附加梁延伸段

壳部分完成后，从壳边界向外延伸至支座位置添加梁单元——这些表示精细加劲区域之外的梁端，建模为普通梁单元（与简化路径相同）。

## 壳到梁过渡：刚性连接（主从）

如果跳过或误解此步骤，很可能导致建模错误。

**为什么需要它**：壳/板节点仅承载平动和面内行为——将梁单元（需要传递力和弯矩）连接到壳边界上的一个节点不会正确地将该力/弯矩分布到整个横截面上。相反，壳模型端截面周围的每个节点都必须通过刚性连接到单个**主**节点（梁单元实际连接的位置）。主节点作为刚体移动，所有边界（"从"）节点被约束跟随它——这正确地将梁的端部力和弯矩分散到整个壳截面上。

**实际操作步骤**：

1. 选择壳模型端截面上的所有节点（例如通过该 X 坐标处的框选，宽度足够窄以仅捕获该平面内的节点）。
2. 在该截面中心创建（或识别）一个节点——这是主节点，梁单元将连接到它。
3. 从从节点列表中移除主节点本身（它不应成为自身的从节点）。
4. 使用主节点和其余从节点创建刚性连接。
5. 对壳模型的另一端重复此操作。

在附加梁延伸段之前，对称地在两端执行此操作——否则力会不对称地施加到壳上，结果将失去意义。

## 支座：使用前确认自由度编码

某些 MCP 工具将固定/自由自由度编码为位字符串（例如 `"1111000"`），而非像 `"fix"` 或 `"pin"` 这样的命名类型。**不要猜测数字的顺序或含义**——错误的猜测会静默产生错误的边界条件，分析仍会运行，只是结果不正确。在使用位字符串支座调用之前：

- 检查工具的 schema/文档以了解每个位置代表什么（通常是 Dx、Dy、Dz、Rx、Ry、Rz，按此顺序，但请确认而非假设——某些规范添加了第 7 位或使用不同的顺序）。
- 如果确实不清楚，先使用一个非常简单的测试模型创建支座，并直接在 Midas Civil NX 中检查结果，而不是将可能错误的边界条件提交到完整模型中。

## 壳到梁模型上的荷载

- **壳区域顶部边缘的节点荷载**：沿相关线条选择节点（例如两个已知点之间板区域的顶部边缘），并施加节点力——这近似将分布压力作为离散点荷载施加在网格节点上。
- **延伸段单元上的梁荷载**：直接对延伸组中的梁单元施加均布荷载，与简化路径相同。

## 常见问题（板/壳特有）

### 拉伸/放样边界处重合节点重复

当从现有边反复向外拉伸时，很容易在已有节点（来自之前的拉伸或放样步骤）的位置意外创建新节点，而不是重用现有节点。这会破坏连接性——网格看起来连续，但单元在该接缝处实际上并未连接。在构建新层后，用 `get_all_nodes` 抽查几个边界节点，确认坐标在应共享的位置没有重复。

### 板法向方向不一致

板单元有方向（单元局部 Z 轴/法向方向），影响应力和压力荷载的解读方式。当在不同方向拉伸时（例如对称的 ±Y 拉伸），确保生成的板在整个模型中不会出现不一致的法向——这可能导致自重或压力荷载在某些板上沿意外方向作用。

### RX 型奇异或巨大的意外位移

如果位移结果异常大或出现类似刚体模态，最可能的原因是刚性连接未实际捕获预期的从节点（例如框选遗漏了一个节点，或主节点意外保留在自己的从节点列表中）。在重新运行分析之前，重新检查用于构建每个刚性连接的选区。

### 单位制不匹配

与简化路径相同，不要假定固定的单位制——确认模型实际使用的单位（某些项目使用 N+m 而非 tonf+m），并在信任报告之前，合理性检查荷载大小和反力是否与手工计算的预期值一致。
