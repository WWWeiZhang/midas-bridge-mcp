# Midas Civil MCP — 钢箱截面建模

使用 Midas Civil MCP 工具建立和分析钢箱截面结构模型。
本技能假设 MCP 服务器已连接。

## 开始之前

首先使用 `get_all_nodes` 检查模型状态。如果存在之前会话的数据，告知用户并询问是否清除（在 Midas Civil NX 中按 Ctrl+N）。

**不要假定固定的单位制。** 不同项目使用不同的单位制（如 tonf+m 或 N+m）。无论工具的默认值是什么，始终在分析后通过反力平衡进行验证：总施加荷载应等于总反力。如果 "TONF" 标签报告的值实际上看起来像牛顿（例如 80,000 而不是 ~8），请指出——即使底层数值内部一致，标签也可能具有误导性。

## 建模路径选择

在执行任何操作之前，确定用户实际需要哪个精度等级——两条路径使用不同的单元类型，不可互换。

**简化路径（折减截面梁模型）**——当以下情况时使用：
- 用户只想对带孔的箱梁进行快速静力校核，不需要精确的孔边局部应力。
- 未提及板/壳单元、加劲环或精确的开孔几何形状。
- 继续使用本文件中的工作流程。

**精细路径（带实际开孔几何形状的板/壳模型）**——当用户提到以下任何内容时使用：
- 板或壳单元、放样或拉伸以构建几何形状
- 开孔的实际圆形/几何形状，而不仅仅是"一个孔"
- 开孔周围的加劲环或加劲板
- 将壳模型连接到梁支座的刚性连接/主从节点

如果是这种情况，**请阅读并遵循 `references/plate-shell-box-girder.md`** 而非下面的步骤——它涵盖了一套根本不同的工具和建模顺序（板放样/拉伸、厚度定义、壳到梁的刚性连接），在此使用仅梁的步骤会严重歪曲开孔周围的真实应力分布。

# 简化建模（折减截面梁方法）

## 模型架构

核心模式为：

```
支座(铰接)     延伸梁        主梁(箱形截面)           延伸梁     支座(滑动)
   △           ━━━          ━━━━━━━━━━━━━━━━          ━━━        △
Node_left    Node_start    ○ ○ ○ (腹板开孔)         Node_end   Node_right
X=-0.5        X=0          X=0.5,1.0,...            X=4.0      X=4.5
```

**关键特性：**
- 腹板带圆孔的主箱梁 → 建模为梁单元，在开孔位置使用折减截面（这是一种近似方法，并非实际开孔几何形状——如果用户需要精确的局部应力，请参见上文的精细路径）
- 两端有梁延伸段 → 使用较小截面的独立单元
- 延伸段与主梁交界处刚性连接
- 支座：一端铰接（Dx+Dy+Dz），一端滑动（Dy+Dz，Dx 自由）
- 荷载：自重 + 上翼缘节点荷载 + 延伸段均布荷载

## 核心工作流程

按此顺序建模。每一步依赖前一步。

### 1. 几何设计（内部步骤，不发送至 Midas）

在进行任何 MCP 调用之前设计几何参数：

- **X** = 纵向（沿梁方向），**Y** = 横向（宽度），**Z** = 竖向
- 主梁：通常为 4m，分为 8 段 × 0.5m
- 延伸梁：两端各 0.5m
- 总跨度：5m（4 + 2×0.5）

合理编排节点编号：

```
主梁节点（沿 X 方向，Y=0，Z=0）：
节点 1: X=0.0    节点 2: X=0.5    节点 3: X=1.0（孔中心）
节点 4: X=1.5    节点 5: X=2.0（孔中心）
节点 6: X=2.5    节点 7: X=3.0（孔中心）
节点 8: X=3.5    节点 9: X=4.0

延伸段节点：
节点 10: X=-0.5  （左侧支座）
节点 11: X=4.5   （右侧支座）
```

开孔位于节点 3、5、7——这些节点相邻的单元使用折减截面。

### 2. 创建材料

首先尝试使用 GB(S) 规范的 `create_steel_material`：
- Q345 → standard="GB(S)", db_name="Q345"

如果数据库查找失败（错误 400），**立即回退** 到 `create_user_steel_material`：
- Q345：E=206,000,000,000 Pa，ν=0.3，重度=78,500 N/m³，质量密度=7,850 kg/m³

始终同时创建两者——如果一个失败，另一个成功。

### 3. 创建截面

需要三种截面：

| 截面编号 | 名称 | 宽度(m) | 高度(m) | 用途 |
|---|---|---|---|---|
| 1 | 箱形主梁截面 | 0.6 | 0.4 | 主箱梁（600×400mm） |
| 2 | 延伸梁截面 | 0.3 | 0.3 | 延伸梁（300×300mm） |
| 3 | 开孔段截面 | 0.6 | 0.36 | 开孔段（高度折减 10%） |

全部使用 `create_solid_rectangle_section` 创建。折减截面（3号）考虑了圆孔对腹板的削弱。并行创建所有三种截面。

### 4. 创建节点

在单个 `create_nodes` 调用中批量创建全部 11 个节点。坐标单位为米。

### 5. 创建梁单元

在本地构建单元列表，然后通过一次 `create_beam_elements` 调用发送：

| 单元 | I→J | 截面 | 用途 |
|---|---|---|---|
| 1 | 1→2 | 1（主梁） | 梁段 |
| 2 | 2→3 | 3（开孔） | 开孔段 |
| 3 | 3→4 | 3（开孔） | 开孔段 |
| 4 | 4→5 | 1（主梁） | 梁段 |
| 5 | 5→6 | 3（开孔） | 开孔段 |
| 6 | 6→7 | 3（开孔） | 开孔段 |
| 7 | 7→8 | 1（主梁） | 梁段 |
| 8 | 8→9 | 1（主梁） | 梁段 |
| 9 | 10→1 | 2（延伸） | 左延伸段 |
| 10 | 9→11 | 2（延伸） | 右延伸段 |

所有单元使用材料 1（Q345）。角度均为 0。

### 6. 创建组

**结构组** 用于逻辑组织：
- `主梁组`：单元 1-8
- `延伸梁组`：单元 9-10
- `开孔段组`：单元 2,3,5,6

**边界组**：`支座组`

**荷载组**：`恒载组`，`活载组`

并行创建所有组。

### 7. 施加支座约束

两个支座，两端各一个：

- **节点 10（左侧）**：`create_supports` 使用 `"pin"`——约束 Dx、Dy、Dz
- **节点 11（右侧）**：`create_supports` 使用 `"roller_x"`——约束 Dy、Dz，释放 Dx

两者均归入边界组 `支座组`。

### 8. 创建荷载工况并施加荷载

**荷载工况：**
- `恒载`（case_type="D"）：仅自重
- `活载`（case_type="L"）：节点荷载 + 均布荷载

**施加荷载：**

| 荷载 | 工具 | 目标 | 值 | 组 |
|---|---|---|---|---|
| 自重 | `apply_self_weight` | Z 方向 | -1.0 | 恒载组 |
| 节点荷载 | `apply_nodal_load` | 节点 2,4,6,8 | Fz=-50,000 N | 活载组 |
| 延伸段均布荷载 | `apply_beam_udl` | 单元 9,10 | GZ=-20,000 N/m | 活载组 |

**荷载组合：**
- `create_load_combination`：1.2×恒载 + 1.4×活载，classification="General"

### 9. 运行分析

`run_static_analysis`——无需参数。

### 10. 提取结果并生成报告

并行提取 **两个荷载工况** 的结果：
- `get_nodal_displacements("恒载")` + `get_nodal_displacements("活载")`
- `get_reactions("恒载")` + `get_reactions("活载")`
- `get_beam_forces("恒载")` + `get_beam_forces("活载")`

## 报告格式

分析报告必须包括以下章节：

### 1. 模型概览
包含材料、截面、尺寸、总跨度的表格。

### 2. 模型结构示意图
显示节点布置、支座、开孔和荷载位置的 ASCII 示意图。

### 3. 荷载工况
包含工况名称、类型和荷载描述的表格。

### 4. 位移结果
每个荷载工况的关键节点表格——跨中最大位移 **加粗** 显示。注意：输出单位为米，为便于阅读转换为毫米。

### 5. 支座反力
包含每个工况的左/右支座和总计的表格。包括组合值（1.2 恒载 + 1.4 活载）。验证平衡：总反力 ≈ 总施加荷载。

### 6. 梁内力
关键截面：I 端、跨中、J 端。显示剪力 Vz 和弯矩 My。标注最大值。

### 7. 截面应力校核
计算并验证：
- 弯曲应力 σ = M/W 与屈服强度对比
- 剪应力 τ = V/A 与抗剪强度对比
- 挠度与 L/400 限值对比

### 8. 结论
总结强度、刚度、开孔效应和对称性验证。

## 常见问题

| 错误 | 原因 | 修复 |
|---|---|---|
| 材料创建 400 | 数据库查找失败 | 回退到 `create_user_steel_material` |
| 结果全为零 | 荷载在会话间丢失 | 重新施加荷载并重新运行 |
| 反力过大（如 39000 tonf） | 输出单位为 N 而非 tonf | 转换：除以 9810 得到 tonf。通过荷载平衡校核验证 |
| 反力不对称 | 节点间距不均或单元缺失 | 检查单元连接和截面分配 |
| 位移异常 | 截面或材料分配错误 | 验证单元创建中的截面 ID 与截面定义匹配 |

## 单位验证检查

获取结果后，始终验证：
1. 自重反力 ≈（总体积 × 密度 × g）→ 对于 4m 箱梁应约为 80 kN
2. 活载反力 ≈（4 × 50 kN + 2 × 0.5m × 20 kN/m）= 220 kN
3. 如果数值以 "TONF" 标注但与 kN 匹配，则以 kN 报告并加注说明
